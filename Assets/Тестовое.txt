Проект должен делатся в Unity 5.6.0f3
Целевая платформа - PC
Стиль кода - стандартный из Resharper(VisualSudio) или Rider(убрать все Warnings)

Фреймворки для импорта:
- StrangeIOC
- NewtonsoftJSON
- UniRx
- BackEndless

Ссылка на настроенный пак плагинов(кроме бекенда):
https://drive.google.com/open?id=0B5acbrJ2PK26dUJ3SzMyZXBsWUU

Задание:

1) Создать пустую сцену с базовым контекстом

2) После клика на мышь(тач) на сцену в месте клика появляется "Примитивная фигура меш"
2а) "Примитивная фигура меш" должна загрузится из ассетбандлов(префаб) - один рандомный из 2-3 асетбандлов (Добавить куб, сферу, капсулу (или другие модели на ваше усмотрение)
2b) Названия префабов должны загрузится из BackEndless (джсонка, которая распарсится и там будут названия префабов)

3) После клика на объект - в модельке должно увеличится число кликов по нему

4) В зависимости от количества кликов объект должнен раскрашиваться в разные цвета (команда которая подписана на евент клика на куб),
количество кликов и цвет должен быть прописан в ScriptableObject который грузится из ресурсов
4а) Для  каждого из типа объектов(куб, сфера, капсула) - свои параметры для цвета и настройки

5) После того как создался объект вытаскивается настройки игры (общая админка) в которой указано количество секунд, по происшествии которых объект будет сам менять цвет (на рандомный вообще, он не должен браться из админки)
Ожидание определенного времени сделать с помощью UniRX (не карутиной)

//хранит состояние кликательного объекта - текущее количество кликов и текущий цвет модели, 
//чтобы не вытягивать его из MeshRenderer или SpriteRenderer или другого рендерера
Модель фигуры(GeometryObjectModel):
- int ClickCount
- Color CubeColor

Админка(ГД баланс для объекта) (GeometryObjectData: ScriptableObject):
- List<ClickColorData> ClicksData;

//элемент массива настроек клика - диапазон кликов для конкретного типа объекта и цвет 
//в который будет окрашиваться 3дрендер объект в зависимости от текущего количества кликов в нем
//Пример заполнения:
//Куб 0-3 кликов Красный, Куб 4-12 зеленый и т.д.
ClickColorData:
- string ObjectType;//тип создаваемого объекта (куб, сфера, капсула)
- int MinClicksCount;
- int MaxClicksCount;
- Color Color;

Админка(ГД баланс самой игры) (GameData: ScriptableObject):
- int ObservableTime;

P.S.: Чтобы ассетбандлы успели загрузится в стартовой команде для async команды надо прописать Retain() - во время старта команды, Release() - во время завершения асинхронной операции команды (клики могут начать обрабатываться раньше чем ассетбандлы прогрузяться и будут NullReferenceException)
P.S.2:
Data - обычно это админка(статические данные - обычно ScriptableObject которые не могут меняться во время игры - CONST)
Model - динамическая рантайм моделька для какогото объекта, иногда сохраняется на диск в виде джсонки или протобафа
